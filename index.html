Project logo

Your wordmark or monogram with a transparent background. Used across your Showcase and on the interactive map, so send these once and we'll use them everywhere.

Accepted formats

.svg (preferred) .png transparent (preferred) .ai .eps

Variants needed

  • Full colour / primary version
  • White (reversed) version, for dark backgrounds
  • Black or charcoal version, for light backgrounds

Note: SVG is ideal for sharp rendering at any size. PNGs must have real transparency (minimum 300px height) — a JPG with a white background will sit as a box on the hero image and the map.

Stock list

A spreadsheet listing every apartment with its configuration, areas, price, and availability. This drives the live pricing and availability shown across your Showcase, so it needs to be accurate and kept up to date.

Accepted formats

.csv (preferred) .xlsx

Use our template

Download the template, fill in one row per apartment, and send it back. Keep the column headings exactly as they are so we can import it cleanly.

Showcase Stock List template (.csv)

Columns we need

  • Level — floor the apartment sits on (e.g. G, 1, 2)
  • Lot No — unique lot identifier (e.g. G01)
  • Configuration — plain-text summary (e.g. 1 Bed 1 Bath 1 PR 1 Study 1 Car)
  • Bed · Bath · Powder Room · Study · Car — a number for each
  • Floorplan Type — matches your floor plan naming (e.g. Type B)
  • Aspect — outlook / orientation (e.g. North)
  • Internal Area · External Area · Total Area — in m²
  • Price — optional; only fill this in if you want the price shown on Showcase
  • Status — e.g. Available, Under offer, Sold

Note: One row per apartment. Leave the Price cell blank for any apartment you'd rather not display a price for. Send an updated stock list any time availability or pricing changes.

Interactive map

Map artwork

The designed neighbourhood map from your creative agency, recropped to 16:9. This is the base we build the interactive version from. It should be the final approved artwork.

Accepted formats

.ai layered (preferred) .pdf .indd
16:9

Note: Layered Illustrator (.ai) is strongly preferred, with all layers intact and fonts supplied separately (see Map fonts below). If supplying an InDesign file, include the packaged document fonts and linked files. Your logo for map overlays is already covered under Project logo in Vision — no need to send it again.

Required

Map fonts

Any custom or licensed typefaces used in the map artwork. We need the actual font files, not just the name, so the map renders correctly on our end.

Accepted formats

.otf (preferred) .ttf (preferred) .woff .woff2

Note: If your map uses a standard system font (e.g. Helvetica, Arial) you don't need to supply it. If you're unsure, include it anyway.

Required

Markers list

Every point of interest you want on the map, grouped by category. Each marker should have a name, and ideally a number that corresponds to a legend on the map. Send via email.

Structure

For example

Dining: 1. Otto Ristorante, 2. Bar Francine, 3. Gum Bistro

Parks: 4. South Bank Parklands, 5. Musgrave Park

Education: 6. Brisbane State High, 7. West End State School

Shopping: 8. West Village, 9. Montague Markets

Note: If your map has numbered pins, make sure the numbers on this list match. If there are no numbered pins, please describe what each point refers to so we know what's what.

Required

Transport routes

The transport lines you want drawn on the map: bus routes, train lines, tram lines, ferry routes, with stops and stations marked. Send via email.

Structure

For example

Train: Cleveland line (South Brisbane, Park Road)

Bus: Routes 60, 192 along Montague Rd

Ferry: CityCat (West End, North Quay terminals)

Tram: Route 96 (St Kilda, Melbourne Central)

Note: If transport routes are already drawn on your map artwork (.ai file), you don't need to supply them separately. Just confirm which routes are included.

A site plan or masterplan of the development, recropped to 16:9. Used to show stages, zones, amenities, and named areas as interactive layers. Let us know if you'd like this included and we'll confirm what's needed.

Accepted formats

  • A render or short video per point of interest, if applicable (WEBP or MP4, 16:9)

Layers list example
